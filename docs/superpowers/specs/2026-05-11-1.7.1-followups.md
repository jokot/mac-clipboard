# MaClip 1.7.1 Follow-ups

Bugs and feature requests captured during 1.7.0 QA. Each item has enough context to be turned into a small spec/plan later.

## Bugs

### B1. Image clip icon: keep stable through OCR/QR extraction
**Problem:** Image clip's source icon (captured at poll time) gets replaced with the `?` placeholder after the user runs "Extract text" or "Extract barcode" on it. The extracted result clip also shows `?`.
**Cause:** OCR/extraction creates a new `ClipboardItem` (likely in `ClipboardListViewModel.promoteOrInsertResult` / `promoteOrAddItem`) without forwarding `sourceBundleID`. The image's `cachedText`/`cachedId` updates also reconstruct the image item without forwarding the new fields.
**Fix sketch:**
- In every `ClipboardListViewModel` method that constructs a new `ClipboardItem` from an existing one (`promoteOrInsertResult`, `promoteOrAddItem`, `updateImageItemCache`, etc.), forward `sourceBundleID`, `isConcealed`, `concealedExpiresAt`.
- For OCR result items: inherit `sourceBundleID` from the parent image clip; add a small "OCR" badge in the row (separate from the source icon) so the user can tell the result came from extraction.

### B2. Image clip icon races with frontmost app
**Problem:** Image clips sometimes show the wrong source app icon â€” it captures whatever app is frontmost at the polling tick (every 0.75s), which can drift between the actual copy event and our poll.
**Decision (from QA):** keep current implementation â€” fixing requires hooking pasteboard change events with stronger timing guarantees (pasteboard owners API, etc.) which is out of scope. Just don't make it worse via B1.

### B3. ConcealedBadge countdown updates only every 5 s
**Problem:** Each `ConcealedBadge` row has its own `Timer.publish(every: 5)`. Display jumps (`5m` â†’ `4m` after 60 s, "30s" lingers for up to 5 s).
**Fix sketch:**
- Move to a single shared timer owned by the VM or via SwiftUI's `TimelineView(.periodic(from:by:))` at 1 s.
- Rows observe the shared `now` rather than running per-row timers.

### F3. "Never auto-clear" option

### F4. Uninstall â†’ fresh state UX
**Problem:** macOS drag-to-trash doesn't clear UserDefaults, so users who uninstall + reinstall don't get a fresh seed. User has to know about AppCleaner or manually delete prefs/AppSupport.
**Fix sketch (option A, cheapest):**
- Write a sentinel file `~/Library/Application Support/MaClip/.seeded` first time the seed is applied.
- In `AppSettings.makeInitialExcludedBundleIDs()`: if sentinel absent, apply seed (regardless of UserDefaults state) and write sentinel.
- User who uses AppCleaner (or deletes Application Support manually) â†’ sentinel gone â†’ fresh seed on next launch. UserDefaults persistence no longer matters.

### F5. "Restore Defaults" button in Privacy tab
Adds a button in Settings â†’ Privacy that re-applies the seed exclusion list and resets concealed toggle + timeout to defaults. Inline workaround for users who don't want to use Terminal or AppCleaner.
