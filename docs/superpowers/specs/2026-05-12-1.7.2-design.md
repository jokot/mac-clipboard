# MaClip 1.7.2 ‚Äî Design

**Date:** 2026-05-12
**Status:** Approved (pending implementation plan)
**Target version:** 1.7.2 (build 10)
**Branch:** feature/1.7.2

## Summary

Five items bundled into one release:

1. **B-Warp: Warp screenshots reach MaClip.** Investigate why Warp-sourced screenshots are silently dropped, then ship the fix. Working hypothesis: Warp sets `org.nspasteboard.AutoGeneratedType`, and our 1.7.0 concealed-UTI detection treats that as secret. The nspasteboard convention treats `AutoGeneratedType` as "machine-generated, not user-typed" ‚Äî not equivalent to "secret". Plan: add one-launch diagnostic, confirm, then remove `AutoGeneratedType` from the concealed-UTI set (keep `ConcealedType` and `com.apple.is-sensitive`).
2. **F1+F3: more concealed-clear options.** Add 5 s / 10 s / 20 s entries plus a "Never" entry to the auto-clear timeout picker. "Never" represented as `concealedClearTimeout = -1`.
3. **F7: auto-dismiss overlay after idle.** Default **ON** at 20 s. New Behavior-tab toggle + timeout picker (5 s / 10 s / 20 s / 30 s / 1 min). Any keyboard, mouse, or scroll interaction inside the overlay resets the idle timer.
4. **F2: reveal masked content.** Eye-icon button on each concealed row, visible only on hover. Clicking reveals the real content for 5 seconds, then re-masks. Pasting via click is unchanged (always pastes real content regardless of reveal state).
5. **ESC dismiss + shortcut-toggle for Settings and Info windows.** Settings (‚åò,) and Info (‚åòI) should mirror the Overlay's behavior: pressing ESC closes the window, and the opening shortcut also closes if the window is already key. Currently only Overlay supports ESC.

Bumped to 1.7.2 (build 10).

## Scope

**In scope:**
- One-shot Warp diagnostic added to `ClipboardMonitor.pollPasteboard` (NSLog the pasteboard `types` list) ‚Äî kept in the first commit only, removed in the fix commit.
- Drop `org.nspasteboard.AutoGeneratedType` from the concealed-UTI set in `ClipboardMonitor` (or whichever UTI the diagnostic identifies).
- Picker entries: 5 s / 10 s / 20 s / 30 s / 1 min / 2 min / 5 min / 10 min / 15 min / 30 min / Never (the new entries are 5/10/20 sec plus Never; the existing 30 s through 30 min stay).

- Bump version to 1.7.2 (build 10).

**Out of scope:**

- Visual redesign of the concealed redaction
- Hover-key-modifier alternative for reveal (Q4 alternative `(c)` rejected)

## Warp diagnostic (one-shot)

Add a single `NSLog("[MaClip] pollPasteboard: types=\(pb.types?.map(\.rawValue) ?? [])")` at the top of `pollPasteboard()` immediately after `lastChangeCount = pb.changeCount`. Run app, take a Warp screenshot, inspect captured types in Console.app or the launched-binary's stdout.

Decision tree:
- If the type list contains only `public.tiff` / `public.png` / similar image UTIs ‚Üí not a concealed-UTI issue; needs deeper investigation (skip the rest of B-Warp for this release, log a follow-up).
- If the type list includes `org.nspasteboard.AutoGeneratedType` ‚Üí confirmed; drop it from the concealed set in the fix commit.
- If the type list includes some other unexpected concealed-marker UTI ‚Üí drop that one instead.

The diagnostic NSLog is removed in the same commit as the actual fix.

`PrivacyTab` picker now reads:

```

5 min

```

"Never" is rendered with `tag(TimeInterval(-1))`. Selecting it sets `concealedClearTimeout = -1`.

Capture site in `ClipboardMonitor.pollPasteboard`:

```swift
let timeout = AppSettings.shared.concealedClearTimeout
let expiry: Date? = isConcealed
    ? (timeout == -1 ? Date.distantFuture : Date().addingTimeInterval(timeout))
    : nil
```

Sweep site in `ClipboardListViewModel.runConcealedExpirySweep`:

```swift
items.removeAll { item in
    guard item.isConcealed,
          let expiry = item.concealedExpiresAt else { return false }
    return expiry <= now && expiry < Date.distantFuture
}
```

Effectively: items with `Date.distantFuture` expiry are never swept. Same condition expresses "Never" without needing a separate field on `ClipboardItem`.

## Auto-dismiss overlay after idle

New settings, surfaced in the Behavior tab:

```swift
@Published var autoDismissOverlayEnabled: Bool { didSet { ‚Ä¶ } }    // default true
@Published var autoDismissOverlayTimeout: TimeInterval { didSet { ‚Ä¶ } }  // default 20
```

Behavior tab gets a new section above the existing pasteOnClick/moveTopOnClick toggles:

```
[ ] Auto-dismiss overlay after idle    [ 20 sec ‚ñæ ]
    Closes the clipboard overlay automatically when there is no
    keyboard, mouse, or scroll activity inside the window.
```

Timeout picker entries: `5 sec`, `10 sec`, `20 sec`, `30 sec`, `1 min`. (No "Never" entry ‚Äî if the user wants no auto-dismiss, they turn the toggle off.)

Implementation in `OverlayWindowController`:

```swift
private var idleTimer: Timer?
private var lastActivity: Date = Date()
private var idleMonitor: Any?

// in show():
//   startIdleTracking()
// in hide() / hideImmediatelyAndPaste() / hideImmediatelyRefocusOnly():
//   stopIdleTracking()

private func startIdleTracking() {
    guard AppSettings.shared.autoDismissOverlayEnabled else { return }
    lastActivity = Date()

    idleMonitor = NSEvent.addLocalMonitorForEvents(
        matching: [.keyDown, .keyUp, .mouseMoved, .leftMouseDown, .scrollWheel]
    ) { [weak self] event in
        self?.lastActivity = Date()
        return event
    }

    idleTimer = Timer.scheduledTimer(withTimeInterval: 1.0, repeats: true) { [weak self] _ in
        guard let self else { return }
        let elapsed = Date().timeIntervalSince(self.lastActivity)
        if elapsed >= AppSettings.shared.autoDismissOverlayTimeout {
            Task { @MainActor in
                self.hideImmediatelyRefocusOnly()
            }
        }
    }
}

private func stopIdleTracking() {
    idleTimer?.invalidate()
    idleTimer = nil
    if let idleMonitor {
        NSEvent.removeMonitor(idleMonitor)
        self.idleMonitor = nil
    }
}
```

Notes:
- Local monitor only fires while MaClip is the active app. The overlay's `level = .floating` keeps it key.
- Timer interval is 1 s ‚Äî coarse enough that overhead is negligible, fine enough that 5-sec timeouts feel responsive.
- `[.mouseMoved]` requires `window.acceptsMouseMovedEvents = true`. Set during `createWindow()`.

## Reveal masked content (eye-button)

New helper:

```swift
private struct RevealButton: View {
    let action: () -> Void
    var body: some View {
        Button(action: action) {
            Image(systemName: "eye")
                .font(.system(size: 12, weight: .medium))
                .foregroundColor(.secondary)
                .frame(width: 16, height: 16)
                .contentShape(Rectangle())
                .help("Reveal for 5 seconds")
        }
        .buttonStyle(.plain)
    }
}
```

Each concealed subview (`TextItemContent`, `URLItemContent`, `ImageItemContent`) gains:

```swift
@State private var revealedUntil: Date? = nil
```

And a derived `isCurrentlyRevealed: Bool` from `revealedUntil`. Render the unmasked preview when `isCurrentlyRevealed`; render `‚Ä¢‚Ä¢‚Ä¢‚Ä¢‚Ä¢‚Ä¢` otherwise.

Show the eye `RevealButton` in the trailing area whenever `isConcealed` AND `isHovered`. Tapping schedules `revealedUntil = Date().addingTimeInterval(5)` and starts a 5 s `Timer.scheduledTimer` (per-row, transient) that flips `revealedUntil = nil` when fired.

Click-to-paste already pastes the real content; the reveal feature is purely visual.

## ESC + shortcut-toggle for Settings and Info windows

Both windows currently:
- Settings: opens with ‚åò,; existing local monitor inside `SettingsWindowController.init` closes on ‚åò,-while-key. No ESC handling.
- Info: opens with ‚åòI (or via the menu); no ESC, no shortcut-toggle today.

Add in `SettingsWindowController.init` ‚Äî replace the existing single-shortcut monitor with this combined one:

```swift
NSEvent.addLocalMonitorForEvents(matching: .keyDown) { [weak self] event in
    guard let self = self, self.window?.isKeyWindow == true else { return event }
    if event.keyCode == 53 {   // Escape
        self.window?.close()
        return nil
    }
    if event.modifierFlags.contains(.command), event.charactersIgnoringModifiers == "," {
        self.window?.close()
        return nil
    }
    return event
}
```

For Info window: locate `InfoWindow` / its controller (verify exact name when implementing). Add the same monitor pattern with `event.keyCode == 53` (Esc) and `event.charactersIgnoringModifiers == "i"` with `.command` (‚åòI).

## Data Model

`ClipboardItem` unchanged in 1.7.2. (Reveal state is per-row UI state, not persisted.)

`AppSettings`:
```swift
@Published var autoDismissOverlayEnabled: Bool        // default true
@Published var autoDismissOverlayTimeout: TimeInterval // default 20
```
UserDefaults keys: `settings.autoDismissOverlay.enabled`, `settings.autoDismissOverlay.timeout`.

## Migration

- Existing histories: nothing to do ‚Äî no new persisted fields.
- New `autoDismissOverlayEnabled` defaults to `true`; pre-1.7.2 users get auto-dismiss on first 1.7.2 launch. Documented in the CHANGELOG. Users who don't want it can disable in Settings ‚Üí Behavior.
- `concealedClearTimeout` previously could not equal `-1`; that value is now reserved for "Never". Existing values (300 default, etc.) unaffected.

## Edge Cases

1. **Auto-dismiss fires while user is selecting via keyboard** ‚Äî local monitor catches `.keyDown` and resets timer; only true idle triggers dismiss.
2. **`autoDismissOverlayTimeout = -1`** ‚Äî picker doesn't expose it; if someone manually sets via UserDefaults, the comparison `elapsed >= -1` is always true. Validate at picker level only.
3. **`Never` concealed timeout: existing concealed items already have an `concealedExpiresAt` set** ‚Äî items captured before user picked "Never" still expire normally. Acceptable; only new items honor the current setting.
4. **Reveal active when overlay auto-dismisses** ‚Äî overlay closes, per-row `@State revealedUntil` is lost (view destroyed). On next open, item is masked again. Correct.
5. **Reveal click triggers `onTapGesture` for paste?** ‚Äî no: `Button` action takes precedence over the parent row's `.onTapGesture` because SwiftUI honors the inner-most hit target. Confirmed pattern.
6. **Warp diagnostic NSLog left in production** ‚Äî gated to be removed in the same commit as the fix. Manual verification in QA.
7. **Settings/Info ESC monitor conflicts with text-field editing inside Settings** ‚Äî Cocoa already handles ESC for text fields (cancels edit). Our monitor only fires when window is key + key is not consumed; for fields like `cleanOlderThanDays`, ESC currently doesn't do much. Acceptable.

## Error Handling

No new failure modes. Local event monitors are idempotent and clean up on `cleanup()` calls.
